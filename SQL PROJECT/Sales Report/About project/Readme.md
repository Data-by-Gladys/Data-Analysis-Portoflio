# Refresh Supermarket Sales Analysis

## Case Study:
Refresh Supermarket is reviewing its sales and customer data to gain insights for operational 
improvements and strategic decision-making. The management wants an overview of customer orders, 
sales performance, and trends for the past period.

## Objective:
1. Leverage SQL skills to extract and analyze data from the Refresh Supermarket database,
   answering business questions and uncovering actionable insights. The analysis uses two tables:
 
   a. Customer Table: contains customer_id, customer_name, country, score.
   
   b. Order Table: contains order_id, customer_id, order_date, sales.

## Project Workflow / Key Questions
The project addresses key business questions that management needs to answer:

 a. Retrieve customer information and sales details
    Get the customer_name, country, and sales amount for all customers who have placed an order.
    
 b. Orders in the first half of 2021
    Identify customers who placed orders between January and June 2021.

 c. Customer orders and sales overview
  Show customer_name, country, order_date, and sales for all orders.

 d. High-value customers

 f. Average sales per order

 g. Top-selling customers

 h. Customer scoring and activity

 i. Average sales per country

 j. Customer orders by country (example: Germany)

 k. Total sales by customer

## Project Setup/ Running the analysis
I used SQL Server Management Studio (SSMS) but MySQL, or other SQL-compatible tools can be used.

* Create the database and tables using your preferred method (e.g., .sql file).

* Load sample or real Refresh Supermarket data into customer and order tables.
  
* Write SQL queries to answer each question listed in the workflow.
